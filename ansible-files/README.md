# ansible-files Directory

This directory contains reference copies of files that should be available to students in the lab environment.

## Files in the Student Workspace

When the lab is deployed, the setup automation (`setup-automation/setup-control.sh`) creates these files in `/home/rhel/ansible-files/` on the control node:

### Pre-created by Setup Automation:

1. **`inventory`** - Ansible inventory file with web and database groups
   - Created by: `setup-control.sh`
   - Used in: Module 01 (explore), Module 02+ (playbook target)
   - **This is the ONLY file students interact with at lab start**

2. **`ansible-navigator.yml`** - Configuration for ansible-navigator
   - Created by: `setup-control.sh`
   - Points to inventory, configures execution environment
   - Students don't need to view or edit this file

### Created by Students During Lab:

2. **`system_setup.yml`** - Comprehensive playbook generated by Lightspeed
   - Created in: Module 02, Task 2
   - Used in: Module 03 (run and verify)
   - Reference copy provided here for testing/validation

3. **`templates/motd.j2`** - Jinja2 template for dynamic MOTD
   - Created in: Module 02, Task 3 (student copy-pastes content from instructions)
   - Used by: `system_setup.yml` template task
   - Reference copy provided here

### Configuration Files:

4. **`ansible.cfg`** - Optional Ansible configuration
   - Points to the inventory file
   - Disables host key checking
   - Note: The main config is in `/home/rhel/.ansible.cfg` (created by setup script)

## File Contents

### inventory
```ini
[web]
node1
node2

[database]
node3

[nodes:children]
web
database
```

### ansible.cfg
```ini
[defaults]
inventory = inventory
host_key_checking = False
```

### templates/motd.j2
```jinja2
Welcome to {{ ansible_hostname }}.
OS: {{ ansible_distribution }} {{ ansible_distribution_version }}
Architecture: {{ ansible_architecture }}
```

### system_setup.yml
See the complete playbook in the file. It includes:
- Variables: user_name, package_name, apache_service_name
- Conditionals targeting the web group
- Handler for firewall reload
- Template deployment for MOTD

## Usage During Lab Development/Testing

When testing the lab locally or in development:

1. Copy the entire `ansible-files` directory to your test control node:
   ```bash
   scp -r ansible-files/ rhel@control:/home/rhel/
   ```

2. Or manually create just the inventory (the rest should be generated by students):
   ```bash
   mkdir -p /home/rhel/ansible-files
   cp ansible-files/inventory /home/rhel/ansible-files/
   ```

## Notes

- At lab start, students see `inventory` (the only file they interact with) and `ansible-navigator.yml` (config, they don't touch it)
- `system_setup.yml` is generated BY Lightspeed in Module 02
- `templates/motd.j2` is created manually BY students (copy-paste from instructions) in Module 02
- The reference copies here are for validation scripts, solve buttons, and testing
- In production deployment, `setup-control.sh` handles creating the initial files
