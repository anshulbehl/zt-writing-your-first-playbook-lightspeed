= 1.8 - Templates

Jinja2 templates let you generate *dynamic configuration files* that are customized per host at runtime. Ansible renders the template on the control node — substituting host-specific facts — and then copies the result to each managed node. The same template file produces a different output on each host.

In this challenge, you will prompt Lightspeed to generate a Jinja2 template for `/etc/motd` (the login message of the day) and the playbook task that deploys it.

=====
.How templates work in Ansible
[%collapsible]
====
* Template files use the `.j2` extension and mix static text with `{{ variable }}` placeholders.
* The `ansible.builtin.template` module renders the file and copies it to the managed node.
* Ansible facts like `ansible_hostname`, `ansible_distribution`, and `ansible_architecture` are automatically available to every template without any extra configuration.
====
=====

== Hands-On Experience

======
.☑️ Task 1 - Generate motd.j2 with Lightspeed
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab.
====

. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Generate a Jinja2 template for /etc/motd that displays the hostname, OS distribution and version, and system architecture using Ansible facts
----
. Review the generated template in the new editor tab. Verify that:

* `{{ ansible_hostname }}` is present
* `{{ ansible_distribution }}` and `{{ ansible_distribution_version }}` are present
* `{{ ansible_architecture }}` is present

[TIP]
====
*Expected Lightspeed output:*

[source,jinja2]
----
Welcome to {{ ansible_hostname }}.
OS: {{ ansible_distribution }} {{ ansible_distribution_version }}
Architecture: {{ ansible_architecture }}
----
====

. Create the *templates* directory in the *ansible-files* directory first. In the VS Code file explorer, right-click *ansible-files* and select *New Folder* → name it `templates`.

. Press `Ctrl+S` and save the generated file as *motd.j2* inside the *ansible-files/templates* directory.
=====
======


======
.☑️ Task 2 - Generate the updated playbook with the template task
[%collapsible]
=====
. Open the Lightspeed generation panel again.
. Enter the following prompt:
+
----
Write a playbook targeting all hosts that creates user padawan, installs and configures httpd and firewalld with an HTTP firewall rule on web servers only with handlers, and deploys a Jinja2 motd template from templates/motd.j2 to /etc/motd on all hosts
----
. Review the generated playbook. Verify that the template task is present:

* Module is `ansible.builtin.template`
* `src: templates/motd.j2`
* `dest: /etc/motd`
* No `when:` condition — the MOTD deploys to all hosts

[TIP]
====
*Expected template task in the playbook:*

[source,yaml]
----
    - name: Deploy MOTD from Jinja2 template
      ansible.builtin.template:
        src: templates/motd.j2
        dest: /etc/motd
----
====

. Press `Ctrl+S` and save over *system_setup.yml* in the *ansible-files* directory.

== Learning Outcomes

* Which Ansible facts are automatically available inside templates
* How to use `ansible.builtin.template` to render and deploy a `.j2` file
* How to prompt Lightspeed to generate both a template file and the playbook task that uses it
