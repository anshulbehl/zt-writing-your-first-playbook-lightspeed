= 1.2 - Run the Playbook!

In this challenge, you will run the Lightspeed-generated playbook using *ansible-navigator*, verify that the user was created on the managed node, and discover one of Ansible's most important concepts: *idempotency*.

== Hands-On Experience

======
.☑️ Task 1 - Run the system_setup.yml playbook
[%collapsible]
=====
[NOTE]
====
Switch to the *Control* tab.
====

. Change into the *ansible-files* directory:
+
[source,shell]
----
cd /home/rhel/ansible-files
----

. Run the playbook with *ansible-navigator*:
+
[source,shell]
----
ansible-navigator run system_setup.yml
----
+
IMPORTANT: The first run may take a minute or two. ansible-navigator pulls the execution environment container image and runs the kernel update task on *node1*.

. Watch the output. You should see a *PLAY RECAP* at the end summarizing the changes made:
+
image::navigator-output.png[Output, 90%, align="center"]

[NOTE]
====
The first task you see in the output is always *Gathering Facts*. Ansible automatically collects system information (OS version, IP addresses, installed packages, etc.) from the managed node at the start of each play. Lightspeed uses this fact data when generating its suggestions.

.☑️ Task 3 - Discover idempotency
[%collapsible]
=====
=====
.What is idempotency?
[%collapsible]
====
*Idempotency* means that running an operation once produces the same result as running it many times. An idempotent playbook only makes changes when something actually needs changing — if the desired state already exists, Ansible skips the task instead of re-applying it.

This is one of Ansible's most powerful properties: you can safely run a playbook repeatedly and it will never break a system that is already in the correct state.
====
=====

. Run the playbook a second time:
+
[source,shell]
----
ansible-navigator run system_setup.yml
----

. Compare the output to the first run. Notice:
+
* Task status changed from *changed* (yellow) to *ok* (green)
* The *PLAY RECAP* shows `changed=0` — Ansible confirmed everything was already in place and made no changes
+
image::idem-output.png[Output, 90%, align="center"]

. Now ask Lightspeed to explain *why* this playbook is idempotent. In the *VS Code* tab, select the full playbook content, right-click, and choose *Ansible Lightspeed → Explain*. Ask a follow-up: "Why is this playbook idempotent?"
