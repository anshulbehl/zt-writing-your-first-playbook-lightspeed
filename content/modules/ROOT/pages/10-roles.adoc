= 1.9 - Roles

Roles are how Ansible automation is *packaged and shared*. Instead of one large playbook with everything in it, a role separates tasks, variables, handlers, and templates into a structured directory tree that can be reused across projects and shared through Ansible Galaxy.

In this final challenge, you will use Ansible Lightspeed to generate the tasks, handler, and a Jinja2 template for an *apache* role, then call that role from a simple deployment playbook.

=====
.How roles are structured
[%collapsible]
====
A role lives in a subdirectory under `roles/`. Each folder has a specific purpose:

[source,text]
----
roles/apache/
├── tasks/main.yml       ← the tasks the role runs
├── handlers/main.yml    ← handlers triggered by role tasks
├── templates/           ← Jinja2 template files
├── vars/main.yml        ← role-specific variables
├── defaults/main.yml    ← default variable values
└── meta/main.yml        ← role metadata
----

A playbook calls a role simply by listing it under `roles:` — no task imports required:

[source,yaml]
----
  roles:
    - apache
----

== Hands-On Experience

======
.☑️ Task 1 - Initialize the apache role with ansible-galaxy
[%collapsible]
=====
[NOTE]
====
Switch to the *Control* tab.
====

. Create the `roles` directory and initialize the `apache` role:
+
[source,role="execute"]
----
cd /home/rhel/ansible-files
----
+
[source,role="execute"]
----
mkdir roles
----
+
[source,role="execute"]
----
ansible-galaxy init --offline roles/apache
----

. Verify the scaffold was created:
+
[source,role="execute"]
----
tree roles
----

.☑️ Task 2 - Define role variables
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab and open *ansible-files/roles/apache/vars/main.yml*.
====

. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Generate role variables for an Apache role with apache_package_name set to httpd and apache_service_name set to httpd
----
. Review the generated content. Press `Ctrl+S` to save it over *roles/apache/vars/main.yml*.

[TIP]
====
*Expected output:*

[source,yaml]
----
---
# vars file for roles/apache
apache_package_name: httpd
apache_service_name: httpd
----

======
.☑️ Task 3 - Generate role tasks with Lightspeed
[%collapsible]
=====
. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Generate Ansible role tasks for an Apache web server role that installs the apache_package_name package, starts and enables the apache_service_name service, installs and starts firewalld, and allows HTTP traffic through the firewall on web servers with a notify to reload firewalld
----
. Review the generated content. Verify that:

* The tasks use `{{ apache_package_name }}` and `{{ apache_service_name }}` (not hardcoded values)
* The firewalld task includes `notify: Reload Firewall`
* All tasks use fully qualified module names

. Press `Ctrl+S` and save over *roles/apache/tasks/main.yml*.

[TIP]
====
*Expected output:*

[source,yaml]
----
---
# tasks file for ansible-files/roles/apache
- name: Install Apache web server
  ansible.builtin.package:
    name: "{{ apache_package_name }}"
    state: present

- name: Ensure Apache is running and enabled
  ansible.builtin.service:
    name: "{{ apache_service_name }}"
    state: started
    enabled: true

- name: Install firewalld
  ansible.builtin.dnf:
    name: firewalld
    state: present

- name: Ensure firewalld is running and enabled
  ansible.builtin.service:
    name: firewalld
    state: started
    enabled: true

- name: Allow HTTP traffic on web servers
  ansible.posix.firewalld:
    service: http
    permanent: true
    state: enabled
  when: inventory_hostname in groups['web']
  notify: Reload Firewall
----
====

.☑️ Task 4 - Generate the role handler with Lightspeed
[%collapsible]
=====
. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Generate an Ansible role handler named Reload Firewall that reloads the firewalld service
----
. Review the output and save over *roles/apache/handlers/main.yml*.

[TIP]
====
*Expected output:*

[source,yaml]
----
---
# handlers file for ansible-files/roles/apache
- name: Reload Firewall
  ansible.builtin.service:
    name: firewalld
    state: reloaded
----
====

======
.☑️ Task 5 - Generate the index.html Jinja2 template with Lightspeed
[%collapsible]
=====
. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Generate an HTML Jinja2 template for Apache index.html that displays a welcome message with the ansible_hostname fact in the title and body
----
. Review the generated HTML template. Save it as *index.html.j2* inside *roles/apache/templates/*.

[TIP]
====
*Expected output:*

[source,html]
----
<html>
<head>
<title>Welcome to {{ ansible_hostname }}</title>
</head>
<body>
<h1>Hello from {{ ansible_hostname }}</h1>
</body>
</html>
----
====

. Now generate the task that deploys this template. Open the generation panel again and enter:
+
----
Generate an Ansible task that deploys the index.html.j2 template to /var/www/html/index.html
----

. Add the generated task to the bottom of *roles/apache/tasks/main.yml* and save.
=====
======


======
.☑️ Task 6 - Generate deploy_apache.yml with Lightspeed
[%collapsible]
=====
. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Write a playbook targeting the web group that deploys the apache role
----
. Review the generated playbook. Verify that `roles: [apache]` is listed under the play.

[TIP]
====
*Expected output:*

[source,yaml]
----
---
- name: Setup Apache Web Servers
  hosts: web
  become: true
  roles:
    - apache
----
====

. Press `Ctrl+S` and save as *deploy_apache.yml* in the *ansible-files* directory.
