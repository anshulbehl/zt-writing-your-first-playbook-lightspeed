= 1.9 - Roles

Roles are a way to *package and reuse* Ansible automation. Instead of building one large playbook with everything in it, a role separates tasks, variables, handlers, templates, and files into a structured directory tree that can be shared and reused across projects.

In this final challenge, you will use Ansible Lightspeed to generate the tasks, handler, and a Jinja2 template for an *apache* role, then call that role from a new deployment playbook.

=====
.How roles are structured
[%collapsible]
====
A role lives in a directory under `roles/`. Each subdirectory has a specific purpose:

[source,text]
----
roles/apache/
├── tasks/main.yml       ← the tasks that the role runs
├── handlers/main.yml    ← handlers triggered by role tasks
├── templates/           ← Jinja2 template files
├── vars/main.yml        ← role-specific variables
├── defaults/main.yml    ← default variable values
└── meta/main.yml        ← role metadata
----

A playbook calls a role simply by listing it under `roles:`:

[source,yaml]
----
  roles:
    - apache
----
====

== Hands-On Experience

======
.☑️ Task 1 - Initialize the apache role
[%collapsible]
=====
[NOTE]
====
Switch to the *Control* tab.
====

. Create the `roles` directory and initialize the `apache` role using *ansible-galaxy*:
+
[source,shell]
----
cd /home/rhel/ansible-files
----
+
[source,shell]
----
mkdir roles
----
+
[source,shell]
----
ansible-galaxy init --offline roles/apache
----

. Verify the structure was created:
+
[source,shell]
----
tree roles
----

======
.☑️ Task 2 - Define role variables
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab and open *ansible-files/roles/apache/vars/main.yml*.
====

. Replace the file contents with:
+
[source,yaml]
----
---
# vars file for roles/apache
apache_package_name: httpd
apache_service_name: httpd
----

======
.☑️ Task 3 - Generate role tasks with Lightspeed
[%collapsible]
=====
[NOTE]
====
Open *ansible-files/roles/apache/tasks/main.yml*.
====

. Clear the existing content and start building the task list. Type each task name below and press Tab to accept Lightspeed's suggestion after each one:
+
[source,yaml]
----
---
# tasks file for ansible-files/roles/apache
----
+
[source,yaml]
----
- name: Install Apache web server
----
+
[source,yaml]
----
- name: Ensure Apache is running and enabled
----
+
[source,yaml]
----
- name: Install firewalld
----
+
[source,yaml]
----
- name: Ensure firewalld is running and enabled
----
+
[source,yaml]
----
- name: Allow HTTP traffic on web servers
----

[TIP]
====
For the last task, Lightspeed should generate the `ansible.posix.firewalld` module with `notify: Reload Firewall` — the same pattern you used in the handlers module.

*Expected tasks/main.yml:*

[source,yaml]
----
---
# tasks file for ansible-files/roles/apache
- name: Install Apache web server
  ansible.builtin.package:
    name: "{{ apache_package_name }}"
    state: present

- name: Ensure Apache is running and enabled
  ansible.builtin.service:
    name: "{{ apache_service_name }}"
    state: started
    enabled: true

- name: Install firewalld
  ansible.builtin.dnf:
    name: firewalld
    state: present

- name: Ensure firewalld is running and enabled
  ansible.builtin.service:
    name: firewalld
    state: started
    enabled: true

- name: Allow HTTP traffic on web servers
  ansible.posix.firewalld:
    service: http
    permanent: true
    state: enabled
  when: inventory_hostname in groups['web']
  notify: Reload Firewall
----
====

. Save the file.
=====
======


======
.☑️ Task 4 - Generate the handler with Lightspeed
[%collapsible]
=====
[NOTE]
====
Open *ansible-files/roles/apache/handlers/main.yml*.
====

. Replace the file contents with the handler name, then let Lightspeed generate the rest:
+
[source,yaml]
----
---
# handlers file for ansible-files/roles/apache
- name: Reload Firewall
----
. Press *Enter*, then *Tab* to accept.

[TIP]
====
Lightspeed generates:
[source,yaml]
----
- name: Reload Firewall
  ansible.builtin.service:
    name: firewalld
    state: reloaded
----
====

. Save the file.
=====
======


======
.☑️ Task 5 - Generate the index.html Jinja2 template with Lightspeed
[%collapsible]
=====
[NOTE]
====
Create a new file at *ansible-files/roles/apache/templates/index.html.j2*.
====

. Type the following and let Lightspeed suggest completions as you go:
+
[source,html]
----
<html>
<head>
<title>Welcome to {{ ansible_hostname }}</title>
</head>
<body>
<h1>Hello from {{ ansible_hostname }}</h1>
</body>
</html>
----

. Save the file.

. Next, open *tasks/main.yml* and add a final task at the end to deploy this template. Type:
+
[source,yaml]
----
- name: Deploy custom index.html from template
----
. Press *Enter*, then *Tab* to accept.

[TIP]
====
*Expected:*
[source,yaml]
----
- name: Deploy custom index.html from template
  ansible.builtin.template:
    src: index.html.j2
    dest: /var/www/html/index.html
----
====

. Save the file.
=====
======


======
.☑️ Task 6 - Generate deploy_apache.yml with Lightspeed
[%collapsible]
=====
[NOTE]
====
In *ansible-files*, create a new file named *deploy_apache.yml*.
====

. Type the play header, then use Lightspeed to complete it:
+
[source,yaml]
----
---
- name: Setup Apache Web Servers using the apache role
  hosts: web
  become: true
  roles:
----
. Press *Enter* after `roles:`, then *Tab* to let Lightspeed suggest the role name.

[TIP]
====
*Expected output:*
[source,yaml]
----
---
- name: Setup Apache Web Servers using the apache role
  hosts: web
  become: true
  roles:
    - apache
----

Lightspeed recognizes the roles directory and the apache role you created, and completes the list automatically.
====

. Save the file.
=====
======

ansible-navigator run deploy_apache.yml
