= 1.5 - Conditionals

Conditionals let you control *which tasks run on which hosts*. Using the `when:` keyword, Ansible evaluates an expression at runtime and skips the task if the condition is false. This is how you manage different server roles with a single playbook.

In this challenge, you will prompt Lightspeed to generate a playbook that installs Apache only on web servers — not on the database server.

=====
.How conditionals work in Ansible
[%collapsible]
====
The most common pattern in this lab is checking inventory group membership:

[source,yaml]
----
when: inventory_hostname in groups['web']
----

`inventory_hostname` is an Ansible *fact* — a variable automatically set to the current host's name. Combined with `groups['web']`, this condition is true only for hosts in the `[web]` inventory group. When it evaluates to false, Ansible logs `skipping` instead of running the task.
====

== Hands-On Experience

======
.☑️ Task 1 - Generate a playbook with conditionals
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab.
====

. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Write a playbook targeting all hosts that creates a user named padawan using a variable called user_name, installs kernel security updates only on web servers, and installs httpd only on web servers using a variable called package_name
----
. Review the generated playbook in the new editor tab. Verify that:

* `hosts: all` targets every host in the inventory
* A `vars:` section defines both `user_name: 'padawan'` and `package_name: httpd`
* The kernel update and httpd install tasks each have `when: inventory_hostname in groups['web']`
* The user creation task has no `when:` — it runs on all hosts

[TIP]
====
*Expected Lightspeed output:*

[source,yaml]
----
---
- name: Basic System Setup
  hosts: all
  become: true
  vars:
    user_name: 'padawan'
    package_name: httpd
  tasks:
    - name: Install security updates for the kernel
      ansible.builtin.dnf:
        name: 'kernel'
        state: latest
        security: true
        update_only: true
      when: inventory_hostname in groups['web']

    - name: Create a new user
      ansible.builtin.user:
        name: "{{ user_name }}"
        state: present
        create_home: true

    - name: Install Apache on web servers
      ansible.builtin.dnf:
        name: "{{ package_name }}"
        state: present
      when: inventory_hostname in groups['web']
----
====

. Press `Ctrl+S` and save over *system_setup.yml* in the *ansible-files* directory.

== Learning Outcomes

By completing this module, you now understand:

* How `when:` conditionals control per-host task execution
* How `inventory_hostname` and `groups['web']` are used to target specific inventory groups
* Why `hosts: all` combined with `when:` is more powerful than separate plays per group
* How to verify conditional behavior by observing `skipping` in the playbook output
