= 1.5 - Conditionals

Conditionals let you control *which tasks run on which hosts*. In this challenge, you will use Ansible Lightspeed to add an Apache installation task that only runs on web servers — not on the database server.

=====
.How conditionals work in Ansible
[%collapsible]
====
The `when:` keyword makes a task conditional. Ansible evaluates the expression and only runs the task if it is true. A common pattern is checking inventory group membership:

[source,yaml]
----
when: inventory_hostname in groups['web']
----

`inventory_hostname` is an Ansible *fact* — a variable that Ansible fills in automatically with the current host's name. Combined with `groups['web']`, this condition is true only for hosts in the *[web]* group.
====

== Hands-On Experience

======
.☑️ Task 1 - Prompt Lightspeed to target all nodes
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab and open *system_setup.yml*.
====

. Update `hosts:` to target all inventory hosts. Change the line to:
+
[source,yaml]
----
  hosts: all
----
+
. Add a `package_name` variable to the `vars:` section. After `user_name: 'padawan'`, type:
+
[source,yaml]
----
    package_name: httpd
----
=====
======


======
.☑️ Task 2 - Use Lightspeed to add a conditional Apache install task
[%collapsible]
=====
. After the user creation task, press Enter and type the following task name:
+
[source,yaml]
----
    - name: Install Apache on web servers only
----
. Press *Enter*, then *Tab* to accept Lightspeed's suggestion.

[TIP]
====
*Expected Lightspeed output:*

[source,yaml]
----
    - name: Install Apache on web servers only
      ansible.builtin.dnf:
        name: "{{ package_name }}"
        state: present
      when: inventory_hostname in groups['web']
----

Because `package_name` is already defined in `vars:` and Lightspeed sees the task name mentions "web servers only", it generates both the DNF task and the `when:` conditional automatically.
====

. Also update the kernel update task — it should only run on web servers too. Find that task and ask Lightspeed to add the conditional. Select the task, right-click → *Ansible Lightspeed → Explain*, then manually add:
+
[source,yaml]
----
      when: inventory_hostname in groups['web']
----
+
. Save the file.
=====

.☑️ Task 3 - Run the playbook
[%collapsible]
=====
[NOTE]
====
Switch to the *Control* tab.
====

[source,shell]
----
cd /home/rhel/ansible-files
----

[source,shell]
----
ansible-navigator run system_setup.yml
----

[NOTE]
====
Watch for *skipping* messages in the output. When a task's `when:` condition is false, Ansible logs `skipping` for that host instead of running the task. This is how you confirm the conditional is working.
