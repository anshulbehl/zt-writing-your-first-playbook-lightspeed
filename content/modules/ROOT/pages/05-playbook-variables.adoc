= 1.4 - Variables

Variables make playbooks flexible and reusable. Instead of hardcoding a username directly in a task, you define it once in a `vars:` section and reference it everywhere with `"{{ variable_name }}"`. When you need to change it, you update a single line.

In this challenge, you will prompt Lightspeed to generate a playbook that uses a variable for the username, changing the user to *padawan*.

== Hands-On Experience

======
.☑️ Task 1 - Generate a playbook with a username variable
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab.
====

. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Write a playbook targeting the web group that installs kernel security updates and creates a user using a variable called user_name set to padawan
----
. Review the generated playbook in the new editor tab. Verify that:

* A `vars:` section is present with `user_name: 'padawan'`
* The user task references the variable as `name: "{{ user_name }}"` rather than a hardcoded string
* Both tasks are still present

[TIP]
====
*Expected Lightspeed output:*

[source,yaml]
----
---
- name: Basic System Setup
  hosts: web
  become: true
  vars:
    user_name: 'padawan'
  tasks:
    - name: Install security updates for the kernel
      ansible.builtin.dnf:
        name: 'kernel'
        state: latest
        security: true

    - name: Create a new user
      ansible.builtin.user:
        name: "{{ user_name }}"
        state: present
        create_home: true
----

By mentioning "a variable called user_name" in the prompt, Lightspeed generates both the `vars:` definition and the `"{{ user_name }}"` reference in the task — the right connection is made automatically.
====

. Press `Ctrl+S` and save over *system_setup.yml* in the *ansible-files* directory.
=====
======

====

== Learning Outcomes

By completing this module, you now understand:

* How to define and reference variables in an Ansible playbook
* Why variables make playbooks reusable across different environments
* How to prompt Lightspeed to generate both the variable definition and its usage in tasks
* The `"{{ variable_name }}"` Jinja2 syntax for referencing variables
