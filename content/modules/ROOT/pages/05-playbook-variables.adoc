= 1.4 - Variables

Variables make playbooks flexible and reusable. Instead of hardcoding values like usernames directly inside tasks, you define them once and reference them everywhere. In this challenge, you will use Ansible Lightspeed to refactor *system_setup.yml* to use a variable for the username, and change the user to *padawan*.

=====
.How variables work in Ansible
[%collapsible]
====
* Variables are defined in a `vars:` section at the play level (or in separate files for larger projects).
* Variable names must start with a letter or underscore and contain only letters, numbers, and underscores.
* Variables are referenced in tasks using double curly braces in quotes: `"{{ variable_name }}"` — Ansible replaces this with the variable's value at runtime.
====
=====

== Hands-On Experience

======
.☑️ Task 1 - Prompt Lightspeed to add a variable
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab and open *system_setup.yml*.
====

. Add a `vars:` section to the play. Position your cursor after the `become: true` line and before `tasks:`, and type:
+
[source,yaml]
----
  vars:
----
. On the next line, type the variable name as a comment-style hint for Lightspeed:
+
[source,yaml]
----
    user_name: 'padawan'
----
+
[TIP]
====
Lightspeed is context-aware. Once `user_name` is defined in `vars:`, it will automatically suggest `"{{ user_name }}"` in any task where a user name parameter is needed.
====

. Now find the *Create a new user* task. Change the `name:` parameter value. Start deleting `myuser` and type `"{{` — watch Lightspeed suggest `"{{ user_name }}"` to complete it.
. Press *Tab* to accept.

======
.☑️ Task 3 - Verify the new user exists
[%collapsible]
=====
. Check *node1*:
+
[source,shell]
----
ssh node1 id padawan
----
+
. Check *node2*:
+
[source,shell]
----
ssh node2 id padawan
----
+
.Both should return:
[source,text]
----
uid=1003(padawan) gid=1003(padawan) groups=1003(padawan)
----

[NOTE]
====
Your updated *system_setup.yml* should look like this:

[source,yaml]
----
---
- name: Basic System Setup
  hosts: web
  become: true
  vars:
    user_name: 'padawan'
  tasks:
    - name: Install security updates for the kernel
      ansible.builtin.dnf:
        name: 'kernel'
        state: latest
        security: true

    - name: Create a new user named myuser
      ansible.builtin.user:
        name: "{{ user_name }}"
        state: present
        create_home: true
----

Now if you ever need to change the username, you update it in one place — the `vars:` section — and it applies everywhere the variable is used in the playbook.
