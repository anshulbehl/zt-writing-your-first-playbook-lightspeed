= 1.7 - Loops

Loops let Ansible repeat a task for a list of values — creating several users in one task instead of writing a separate task for each. In this challenge, you will prompt Lightspeed to generate a new playbook that creates three users using a `loop`.

=====
.How loops work in Ansible
[%collapsible]
====
The `loop:` keyword provides a list of items to iterate over. The special variable `{{ item }}` is replaced with each list value in turn:

[source,yaml]
----
  tasks:
    - name: Create a new user
      ansible.builtin.user:
        name: "{{ item }}"
        state: present
      loop:
        - alice
        - bob
        - carol
----

Ansible runs this single task three times — once per name. The output shows three sub-results listed under one task name.
====
=====

== Hands-On Experience

======
.☑️ Task 1 - Generate loop_users.yml with Lightspeed
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab.
====

. Open the Lightspeed generation panel.
. Enter the following prompt:
+
----
Write a playbook targeting node1 that creates users alice bob and carol each with a home directory using a loop
----
. Review the generated playbook in the new editor tab. Verify that:

* `hosts: node1` is set
* A single `ansible.builtin.user` task uses `name: "{{ item }}"` as the loop variable
* The `loop:` block lists `alice`, `bob`, and `carol`

[TIP]
====
*Expected Lightspeed output:*

[source,yaml]
----
---
- name: Create multiple users with a loop
  hosts: node1
  become: true
  tasks:
    - name: Create users alice bob and carol with a loop
      ansible.builtin.user:
        name: "{{ item }}"
        state: present
        create_home: true
      loop:
        - alice
        - bob
        - carol
----

Lightspeed extracts the names from your prompt and builds both the `loop:` list and the `{{ item }}` reference automatically.
====

. Press `Ctrl+S` and save the file as *loop_users.yml* in the *ansible-files* directory.
=====
======


======
.☑️ Task 2 - Run loop_users.yml
[%collapsible]
=====
[NOTE]
====
Switch to the *Control* tab.
====

[source,role="execute"]
----
cd /home/rhel/ansible-files
----

[source,role="execute"]
----
ansible-navigator run loop_users.yml
----

[NOTE]
====
The task appears *once* in the output, but Ansible runs it *three times* — once per item. You will see three sub-results listed beneath the single task name, one for each username.

* How to read loop sub-results in ansible-navigator output
