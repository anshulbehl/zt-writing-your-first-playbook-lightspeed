= 1.7 - Loops

Loops let Ansible repeat a task multiple times with different values — for example, creating several users in a single task instead of writing one task per user. In this challenge, you will use Ansible Lightspeed to generate a new playbook that creates three users using a loop.

=====
.How loops work in Ansible
[%collapsible]
====
The `loop:` keyword provides a list of items. The special variable `{{ item }}` is replaced with each list value in turn. Instead of three separate user tasks, you write one task with a loop:

[source,yaml]
----
  tasks:
    - name: Create a new user
      ansible.builtin.user:
        name: "{{ item }}"
        state: present
      loop:
        - alice
        - bob
        - carol
----

Ansible runs this task three times — once for each name in the list.
====

== Hands-On Experience

======
.☑️ Task 1 - Create loop_users.yml with Lightspeed
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab.
====

. In the *ansible-files* directory, right-click and create a new file named *loop_users.yml*.
. Add the play header:
+
[source,yaml]
----
---
- name: Create multiple users with a loop
  hosts: node1
  become: true
  tasks:
----

. Now type the following task name and let Lightspeed generate the loop:
+
[source,yaml]
----
    - name: Create users alice bob and carol using a loop
----
. Press *Enter*, then *Tab* to accept the suggestion.

[TIP]
====
*Expected Lightspeed output:*

[source,yaml]
----
    - name: Create users alice bob and carol using a loop
      ansible.builtin.user:
        name: "{{ item }}"
        state: present
        create_home: true
      loop:
        - alice
        - bob
        - carol
----

Lightspeed extracts the names from your task description and builds the `loop:` list. The `{{ item }}` placeholder is the key pattern — it is replaced by each list entry at runtime.
====

. Save the file.
=====
======


======
.☑️ Task 2 - Run the loop_users.yml playbook
[%collapsible]
=====
[NOTE]
====
Switch to the *Control* tab.
====

[source,shell]
----
cd /home/rhel/ansible-files
----

[source,shell]
----
ansible-navigator run loop_users.yml
----

[NOTE]
====
Watch the output. The task appears *once* in the play output, but Ansible runs it *three times* — once for each item. You will see three sub-results listed under the single task name.
