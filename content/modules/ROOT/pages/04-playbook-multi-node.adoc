= 1.3 - From One to Many

The real power of automation is managing *many systems at once*. In this challenge, you will use Ansible Lightspeed to update your playbook to target the *web* group — both *node1* and *node2* — instead of just a single host.

== Hands-On Experience

======
.☑️ Task 1 - Ask Lightspeed to target the web group
[%collapsible]
=====
[NOTE]
====
Switch to the *VS Code* tab and open *system_setup.yml*.
====

. Lightspeed can refactor existing playbooks. Place your cursor on the `hosts: node1` line and select that line.
. Right-click and choose *Ansible Lightspeed → Explain* to confirm Lightspeed understands the current state of the file.
. Now manually update the `hosts:` line to target the *web* inventory group:
+
[source,yaml]
----
- name: Basic System Setup
  hosts: web
  become: true
----
+
[TIP]
====
You can also place your cursor right after `hosts: ` and delete `node1`, then start typing. Lightspeed may offer a completion suggesting group names from the inventory — try it!
====

. Save the file with *Ctrl+S*.
=====
======


======
.☑️ Task 2 - Run the updated playbook
[%collapsible]
=====
[NOTE]
====
Switch to the *Control* tab.
====

. Run the playbook again:
+
[source,shell]
----
cd /home/rhel/ansible-files
----
+
[source,shell]
----
ansible-navigator run system_setup.yml
----
+
WARNING: This run may take a minute or two — *node2* is being configured for the first time, so it needs to download metadata and run the kernel update task.
=====
======


======
.☑️ Task 3 - Verify myuser exists on both nodes
[%collapsible]
=====
. Check *node1*:
+
[source,shell]
----
ssh node1 id myuser
----

. Check *node2*:
+
[source,shell]
----
ssh node2 id myuser
----
+
.Both should return output similar to:
[source,text]
----
uid=1002(myuser) gid=1002(myuser) groups=1002(myuser)
----

[NOTE]
====
By simply changing `hosts: node1` to `hosts: web`, the playbook now runs on every host in the *[web]* group. If you later add more web servers to the inventory, you do not need to change the playbook — it will automatically include the new hosts.
====
