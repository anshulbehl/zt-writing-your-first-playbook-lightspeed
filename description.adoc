= Writing Your First Playbook with Ansible Lightspeed

== Lab Overview

This streamlined, 4-module lab showcases *Ansible Lightspeed's VS Code extension* for generating comprehensive playbooks from plain-English prompts. Students learn LLM prompting best practices by constructing a single comprehensive prompt rather than iterating multiple times.

*Duration:* ~60-90 minutes

*Difficulty:* Beginner

== What You'll Learn

* How to construct effective LLM prompts for playbook generation
* How Ansible Lightspeed generates production-ready playbooks from plain English
* How to verify comprehensive automation (variables, conditionals, handlers, loops, templates)
* Understanding idempotency and multi-host targeting

== Lab Modules

=== Module 1: Meet Ansible Lightspeed
* Verify Lightspeed connection in VS Code
* Explore the pre-configured inventory
* Understand the lab environment (control node + 3 managed nodes)

=== Module 2: Generate a Comprehensive Playbook
* Construct a detailed prompt that generates a full playbook with:
** Variables
** Conditionals
** Handlers
** Templates
* Create a Jinja2 template for the Message of the Day
* Use Lightspeed's explain feature

=== Module 3: Run and Verify
* Execute the comprehensive playbook with ansible-navigator
* Verify all components across multiple hosts:
** User creation
** Package installation (conditional on host groups)
** Service management
** Firewall configuration
** Template deployment
* Discover idempotency by running the playbook twice

=== Module 4: Wrap-Up and Next Steps
* Summary of concepts covered
* Resources for continued learning

== Lab Environment

*Infrastructure:*

* 1 control node (RHEL 9.6, 8G RAM, 2 cores)
** Pre-configured with ansible-navigator and execution environment
** Inventory already set up
* 3 managed nodes (RHEL 9.6, 8G RAM each, 2 cores each)
** node1, node2: web servers group
** node3: database servers group
* 1 VS Code VM (8G RAM, 2 cores)
** Browser-based VS Code with Ansible extension
** Ansible Lightspeed integration via LiteLLM proxy

*Services:*

* VS Code: Web-based IDE with Ansible Lightspeed
* Control Terminal: SSH access to control node for running playbooks

== Prerequisites

* Basic understanding of YAML syntax
* Familiarity with SSH and terminal commands
* No prior Ansible experience required

== Design Philosophy

This lab was designed to:

* *Reduce token consumption* - Generate the complete playbook once instead of iterating multiple times
* *Teach LLM prompting* - Focus on constructing comprehensive prompts with clear requirements
* *Showcase Lightspeed* - Demonstrate the power of generating production-ready automation from plain English
* *Streamline the learning path* - 4 focused modules instead of 10, covering the same concepts more efficiently

The comprehensive playbook approach works because Lightspeed's playbook generation feature creates complete, production-ready playbooks. By teaching students to construct a complete prompt upfront, we demonstrate best practices for AI-assisted automation.

== Target Audience

* IT professionals new to Ansible
* Automation engineers exploring AI-assisted tooling
* System administrators looking to accelerate playbook development
* Anyone interested in LLM-powered infrastructure automation

== Learning Outcomes

After completing this lab, students will be able to:

* Use Ansible Lightspeed to generate comprehensive playbooks from natural language
* Construct effective prompts that specify all required automation components
* Understand and verify idempotent playbook execution
* Use conditionals to target specific host groups
* Deploy templates using Jinja2
* Run playbooks with ansible-navigator and execution environments
