= zt-writing-your-first-playbook
Zero Touch Writing your first playbook lab - Streamlined for Ansible Lightspeed


== Lab Overview

This is a condensed, 4-page lab focused on showcasing *Ansible Lightspeed's VS Code extension* for generating comprehensive playbooks from plain-English prompts. The lab teaches LLM prompting best practices by having students construct a single comprehensive prompt rather than iterating multiple times.

*Key Learning Objectives:*

* How to construct effective LLM prompts for playbook generation
* How Ansible Lightspeed generates production-ready playbooks from plain English
* How to verify comprehensive automation (variables, conditionals, handlers, loops, templates)
* Understanding idempotency and multi-host targeting


== Lab Structure

The lab consists of 4 modules:

1. *Meet Ansible Lightspeed* - Introduction, verify connection, explore inventory
2. *Generate a Comprehensive Playbook* - Construct a detailed prompt that generates a full playbook with variables, conditionals, handlers, and templates in one shot
3. *Run and Verify* - Execute the playbook and verify all components across multiple hosts
4. *Wrap-Up and Next Steps* - Summary and future learning paths

      solveButton: true
    - name: 02-generate-comprehensive-playbook
      label: Generate a Comprehensive Playbook
      solveButton: true
    - name: 03-playbook-run-it
      label: Run and Verify
      solveButton: true
    - name: 04-wrap-up
      label: Wrap-Up and Next Steps
      solveButton: false             # No solve button for wrap-up

tabs:
  - name: VS Code                     # VS Code with Lightspeed extension
    url: https://vscode-${guid}.${domain}/
    external: false
  - name: Control                     # Terminal for ansible-navigator
    url: /wetty
    external: false
----

NOTE: This structure works with the agnosticd role *showroom*.


== Design Philosophy

This lab was redesigned to:

* *Reduce token consumption* - Generate the complete playbook once instead of iterating multiple times
* *Teach LLM prompting* - Focus on constructing comprehensive prompts with clear requirements
* *Showcase Lightspeed* - Demonstrate the power of generating production-ready automation from plain English
* *Streamline the learning path* - 4 pages instead of 10, covering the same concepts more efficiently

The comprehensive playbook approach works because Lightspeed's playbook generation feature cannot edit existing playbooks - it only generates new ones. By teaching students to construct a complete prompt upfront, we avoid the iteration problem and demonstrate best practices for AI-assisted automation.


== Key Changes from Original Lab

* *Compressed modules 02-09 into a single module* (02-generate-comprehensive-playbook)
* *Removed roles module* (10) - complexity not needed for a Lightspeed demo
* *Enhanced verification* (module 03) to validate all playbook components across multiple hosts
* *Streamlined runtime automation* from 10 directories to 4
* *Added LLM prompting guidance* - explicit instruction on constructing comprehensive prompts
