# Reference Files for Lab

## Overview

This document explains which files should be available to students at different stages of the lab, and where reference copies are stored for validation/testing.

## Files Created by Setup Automation

The `setup-automation/setup-control.sh` script runs during lab deployment and creates these files on the control node at `/home/rhel/`:

### 1. `.ansible.cfg` (in home directory)
```ini
[defaults]
inventory = /home/rhel/ansible-files/inventory
host_key_checking = False
```

### 2. `.ansible-navigator.yml` (in home directory and working directory)
- Configures ansible-navigator execution environment
- Points to inventory file
- Sets container engine to podman
- Configures logging

### 3. `ansible-files/inventory`
```ini
[web]
node1
node2

[database]
node3

[nodes:children]
web
database
```

**This is the ONLY file in `/home/rhel/ansible-files/` that students interact with at lab start.**

Students see it in Module 01 when they explore the pre-created inventory.

### 4. `ansible-files/ansible-navigator.yml`
Configuration for ansible-navigator (points to inventory, sets execution environment).
Students don't need to view or edit this file - it's created by setup automation and works in the background.

## Files Created by Students During the Lab

### Module 02: Generate Comprehensive Playbook

**Task 2: Generate playbook**
- Students use Lightspeed to generate `system_setup.yml`
- They save it to `/home/rhel/ansible-files/system_setup.yml`
- Reference copy: `ansible-files/system_setup.yml` (this repo)

**Task 3: Create template**
- Students manually create `templates/motd.j2` by copy-pasting content from instructions
- Path: `/home/rhel/ansible-files/templates/motd.j2`
- Reference copy: `ansible-files/templates/motd.j2` (this repo)

### Module 03: Run and Verify

Students run the playbook they created in Module 02. No new files are created.

## Reference Files in This Repository

Location: `ansible-files/` directory

```
ansible-files/
├── README.md              # Documentation
├── .gitignore             # Keep all reference files
├── inventory              # Pre-created by setup (students interact with this)
├── ansible-navigator.yml  # Pre-created by setup (students don't touch this)
├── ansible.cfg            # Optional config (for local testing only)
├── system_setup.yml       # Generated by Lightspeed (Module 02)
└── templates/
    └── motd.j2           # Created by student via copy-paste (Module 02)
```

### Purpose of Reference Files

1. **Testing**: Developers can copy the entire `ansible-files` directory to test the lab
2. **Validation**: Validation scripts compare student-generated files against these references
3. **Solve Button**: Solve playbooks copy these files to the student workspace
4. **Documentation**: Shows what the final structure should look like

## Lab Workflow vs File Creation

| Module | What Exists | What Gets Created |
|--------|-------------|-------------------|
| 01 - Inventory | `inventory`, `ansible-navigator.yml` | Nothing |
| 02 - Generate | `inventory`, `ansible-navigator.yml` | Students create: `system_setup.yml`, `templates/motd.j2` |
| 03 - Run & Verify | All files from Module 02 | Nothing (just runs playbook) |
| 04 - Wrap-up | All files from Module 02 | Nothing (informational only) |

## Common Issues and Solutions

### Issue: "inventory file not found"
**Cause**: Setup automation didn't run or failed  
**Solution**: Check that `setup-control.sh` completed successfully during deployment

### Issue: "templates/motd.j2 not found" when running playbook
**Cause**: Student skipped Module 02 Task 3  
**Solution**: Create the template file or click Solve button

### Issue: Validation fails in Module 02
**Cause**: Student's playbook missing required sections  
**Solution**: Check validation script output - it tests for vars, conditionals, handlers, template task

## For Lab Developers

When testing locally:

1. **Start fresh** - Delete `/home/rhel/ansible-files/` if it exists
2. **Run setup** - Execute `setup-control.sh` to create inventory
3. **Follow the lab** - Generate files as a student would
4. **OR copy references** - Copy `ansible-files/*` to `/home/rhel/ansible-files/` for quick testing

When updating the lab:

1. **Update reference files** - Keep `ansible-files/` in sync with what students should generate
2. **Update solve playbooks** - Runtime automation should match reference files
3. **Update validation** - Validation scripts should check for correct structure/content
